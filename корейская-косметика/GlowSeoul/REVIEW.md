# Отчёт Критика / QA-аналитика: сайт GlowSeoul

**Дата ревью:** 21 марта 2026
**Проект:** Интернет-магазин корейской косметики GlowSeoul
**Файлы:** index.html, style.css, script.js
**Сверка с:** research-анализ-конкурентов.md

---

## Оценка по критериям (шкала 1-10)

### 1. Дизайн и визуальный стиль -- 8/10

**Что хорошо:**
- Минималистичный дизайн полностью соответствует стандарту K-beauty магазинов (Soko Glam, Olive Young, Hey!Babes). Белый фон, один акцентный цвет (#e8458b -- hot pink), sans-serif шрифт Inter.
- Грамотная цветовая система через CSS-переменные: primary, primary-light, primary-dark, оттенки серого.
- Визуальная иерархия выдержана: announcement bar -> header -> hero -> секции.
- Hero-секция с градиентным фоном и анимированными плавающими элементами создаёт живое первое впечатление.
- Quiz-секция с тёмным градиентным фоном (#1a1a2e -> #16213e -> #0f3460) выглядит премиально и контрастирует с остальной страницей.

**Что снижает оценку:**
- Вместо реальных фотографий товаров используются emoji-заглушки. Для production это неприемлемо; для прототипа -- допустимо, но research показывает, что "продуктовое фото -- центр дизайна" у всех лидеров.
- Нет цветовых swatches и вариаций на карточках товаров (есть у YesStyle).
- Instagram-секция -- только emoji-заглушки без реальной интеграции.

---

### 2. UX/UI -- 7/10

**Что хорошо:**
- Sticky header с backdrop-filter blur -- современный паттерн, используют лидеры.
- Навигация содержит ключевые точки входа: Каталог, Бренды, 10-Step Routine, Подбор ухода, Блог, Контакты.
- Карточка товара содержит все must-have элементы из research: бренд, название, цена (старая + новая), процент скидки, рейтинг со звёздами, количество отзывов, бейджи (New, Хит, скидка), кнопка wishlist, quick view, кнопка "В корзину".
- Cart Drawer (боковая корзина) -- реализован по лучшему паттерну (Soko Glam, Kiyomi.kz).
- Search overlay с популярными запросами (тегами) -- хороший UX-паттерн.
- Фильтры товаров по категориям (Все / Хиты / Новинки / Со скидкой).
- Секция "Подберите по проблеме кожи" (concern pills) -- ключевой паттерн K-beauty, который research выделяет как must-have.
- Announcement bar с промокодом и порогом бесплатной доставки.

**Что снижает оценку:**
- Поиск не является предиктивным (predictive search). Research отмечает это как Приоритет 1 у Skinorea, YesStyle, Olive Young. Текущий search overlay -- статичная заглушка без реального функционала.
- Фильтры на странице каталога примитивны (4 кнопки). Нет фильтрации по типу кожи, по ингредиентам, по бренду, по цене -- а это must-have по данным research (Olive Young, Hey!Babes).
- Concern pills (проблемы кожи) -- не ведут на отфильтрованные результаты, все ссылки "#".
- Quick view кнопка присутствует на карточках, но не имеет функционала (нет модального окна).
- Wishlist кнопка в header показывает badge "3", но нет страницы/drawer избранного.
- Корзина захардкожена (статичный HTML), кнопки +/- не работают, нет удаления товара, нет пересчёта итога.
- Нет breadcrumb-навигации (есть у BeautyNetKorea).
- Нет страницы отдельного товара (PDP -- Product Detail Page).

### 3. Контент-стратегия -- 8/10

**Что хорошо:**
- Полноценная секция блога "GlowSeoul Journal" с 4 статьями: гайд, разбор ингредиента, обзор, советы. Это соответствует лучшим практикам Soko Glam (The Klog) и Olive Young (OY Magazine).
- Секция "10-Step Korean Skincare Routine" -- research прямо называет это must-have контентом. Реализация с пронумерованными шагами и описаниями -- хорошая.
- Quiz / подбор ухода -- research отмечает это как незанятую нишу среди конкурентов и "сильнейший инструмент для конверсии". Наличие этой секции -- серьёзное конкурентное преимущество.
- Секция преимуществ (доставка, оригинал, консультация, бонусная программа) -- доносит ключевые USP.
- Бренд-маркиз с названиями 12 реальных корейских брендов.

**Что снижает оценку:**
- Статьи блога -- заглушки без реального контента (нет страниц статей).
- Нет кураторских подборок ("Работают, пока вы спите", "Хиты против расширенных пор") -- сильный паттерн Hey!Babes и Soko Glam.
- Нет видеоконтента (Watch & Shop -- есть у Soko Glam и BeautyNetKorea).
- Нет раздела Awards / рейтинги года (Soko Glam Best of Beauty Awards).

### 4. Адаптивность -- 8/10

**Что хорошо:**
- Три медиа-брейкпоинта: 1024px, 768px, 480px -- покрывают tablet и mobile.
- Meta viewport корректно задан.
- Бургер-меню реализовано для мобильных с выпадающей навигацией.
- Сетки грамотно перестраиваются: products grid 4->3->2 колонки, categories 6->3->2, blog 4->2->1.
- Cart drawer на мобильном занимает 100% ширины (правильный паттерн).
- Hero-секция перестраивается в column layout на mobile, заголовок уменьшается 52px->32px->28px.
- Quiz card переходит в column layout.
- Кнопки на 480px становятся full-width.

**Что снижает оценку:**
- Подход не является mobile-first. CSS написан от desktop к mobile (media queries max-width), а research настоятельно рекомендует mobile-first (Soko Glam, YesStyle). Это стилистический недостаток, хотя функционально результат схожий.
- Нет отдельных баннеров/изображений для mobile/desktop (практика Soko Glam).
- Нет touch-specific оптимизаций (swipe для каруселей, touch-friendly target sizes для мелких элементов).
- Бренд-маркиз (бегущая строка) может работать некорректно на узких экранах -- нет специальной адаптации.

### 5. Функциональность JS -- 6/10

**Что хорошо:**
- Бургер-меню: toggle навигации.
- Search overlay: открытие/закрытие с фокусом на input и закрытие по Escape.
- Cart drawer: открытие/закрытие с overlay и блокировкой scroll body.
- Фильтрация товаров с fade-анимацией (opacity transition).
- Countdown-таймер: рассчитывается до конца дня, обновляется каждую секунду.
- Wishlist toggle: переключение fill/stroke SVG-сердечка.
- Scroll-анимации через IntersectionObserver: элементы появляются при скролле (fade-in + translateY).
- Header shadow при скролле.
- Smooth scroll для якорных ссылок.
- Add to cart animation: визуальный feedback "Добавлено" с зелёным фоном.
- Quiz start animation: поочерёдная подсветка шагов.

**Что снижает оценку:**
- Корзина полностью статична: добавление товара не обновляет содержимое drawer, не пересчитывает сумму, не обновляет badge count. Кнопки +/- в корзине не работают. Нет удаления товара.
- Quiz завершается alert() -- крайне примитивно для "сильнейшего инструмента конверсии". Нет реальных вопросов, нет логики подбора, нет результата с рекомендованными товарами.
- Поиск не работает: нет фильтрации/поиска по товарам, нет подсказок.
- Quick View кнопка не привязана ни к какому обработчику.
- Wishlist не синхронизирован: toggle на карточке не обновляет badge "3" в header.
- Smooth scroll: `e.preventDefault()` на всех `a[href^="#"]` может ломать ссылки, если href="#" (пустой якорь) -- страница прокрутится наверх.
- Нет lazy-loading изображений (когда появятся реальные фото, это станет проблемой).

### 6. SEO-готовность -- 5/10

**Что хорошо:**
- Семантические теги: `<header>`, `<nav>`, `<section>`, `<article>`, `<footer>`.
- `<title>` задан: "GlowSeoul -- Корейская косметика".
- `lang="ru"` на html.
- `charset="UTF-8"` и `viewport` мета-теги.
- `aria-label` на кнопках без текста (поиск, избранное, корзина, бургер).
- Заголовки h1-h4 используются с разумной иерархией.

**Что снижает оценку:**
- Нет `<meta name="description">` -- критичный недочёт для SEO.
- Нет Open Graph мета-тегов (og:title, og:description, og:image) -- необходимы для шаринга.
- Нет structured data / JSON-LD (Schema.org Product, BreadcrumbList, Organization). HollyShop из research использует микроразметку Schema.org.
- Нет `<meta name="keywords">`.
- Нет canonical URL.
- Нет alt-атрибутов (нет img-тегов, т.к. emoji вместо изображений -- но при замене на реальные фото alt будет критически важен).
- Нет favicon / apple-touch-icon.
- Все ссылки ведут на "#" -- нет реальной URL-структуры для индексации.
- `<h2>` используется многократно на странице -- это допустимо по HTML5, но для SPA-style лендинга нет отдельных URL для секций.
- Нет sitemap.xml, robots.txt.
- Шрифт загружается через Google Fonts без `preload` -- может замедлять FCP.

### 7. Конкурентные преимущества -- 7/10

**Что хорошо (относительно казахстанского рынка):**
- Quiz для подбора косметики -- research подтверждает, что ни один конкурент этого не имеет. Даже в текущем заглушечном виде это правильное стратегическое решение.
- 10-Step Routine гайд прямо на главной -- must-have контент, который позиционирует магазин как эксперта.
- Фильтрация по проблемам кожи (concern pills) -- есть только у Olive Young, Soko Glam и Hey!Babes.
- Flash Sale с countdown -- создание urgency, как у Jolse и Olive Young.
- Блог-секция -- ни один казахстанский конкурент (Kiyomi.kz, MadeInKorea.kz) не имеет блога.
- Cart drawer -- из казахстанских конкурентов есть только у Kiyomi.kz.
- Цены в тенге (KZT) -- локализация для казахстанского рынка.

**Что снижает оценку:**
- Нет мультиязычности (казахский + русский + английский) -- research выделяет это как Приоритет 2.
- Нет мультивалютности.
- Нет программы лояльности с реальной механикой (упоминается в advantages, но нет реализации).
- Нет сравнения товаров (есть у Hey!Babes).
- Нет прогресс-бара распроданности (есть у YesStyle).
- Нет интеграции с Kaspi (кроме упоминания в footer) -- для Казахстана это критически важно.

### 8. Код-качество -- 8/10

**Что хорошо:**
- CSS-переменные для всех ключевых значений: цвета, радиусы, тени, переходы, шрифт.
- BEM-подобная методология: `.product-card__img`, `.product-card__badge--sale`, `.cart-drawer__header`, `.hero__content`.
- Чёткая структура CSS: reset -> компоненты -> responsive (сверху вниз).
- HTML структурирован комментариями по секциям.
- JS организован по функциональным блокам с комментариями.
- Грамотное использование CSS Grid и Flexbox.
- `aspect-ratio` для карточек товаров.
- `-webkit-line-clamp` для обрезки текста в карточках.
- IntersectionObserver вместо scroll-based анимаций.
- `backdrop-filter: blur()` для header и search overlay.
- `scroll-behavior: smooth` на html.
- Минимальное количество inline-стилей (только динамические через JS).

**Что снижает оценку:**
- BEM не строгий: встречаются селекторы типа `.category-card h3`, `.category-card p`, `.advantage h3` -- смешивание BEM с каскадом.
- В CSS есть дублирование: `.product-card__badge--best` содержит `top: auto; top: 12px;` (двойное присвоение top), строка 427. Аналогично строка 450.
- JS: hardcoded цвет `#e8458b` в wishlist toggle (строка 112) -- должен быть привязан к CSS-переменной или data-атрибуту.
- JS: hardcoded цвет `#4caf50` в add-to-cart animation.
- Нет минификации / бандлинга (ожидаемо для прототипа, но стоит отметить).
- Нет использования CSS custom properties в media queries (все значения захардкожены).
- `font-display: swap` не указан явно (хотя `display=swap` есть в URL Google Fonts).
- Один большой CSS-файл без модульности (для production лучше разделить или использовать препроцессор).

## ТОП-5 сильных сторон

1. **Стратегически верная структура страницы.** Сайт включает все ключевые секции, которые research выделяет как must-have и конкурентные преимущества: quiz для подбора, 10-Step Routine, фильтрация по проблемам кожи, блог, flash sale с таймером, cart drawer. Для казахстанского рынка это значительно выше уровня конкурентов (Kiyomi.kz, MadeInKorea.kz).

2. **Чистый, профессиональный дизайн.** Минимализм с одним акцентным цветом, грамотная типографика, card-based layout, CSS-переменные -- полностью соответствует паттернам лидеров (Soko Glam, Olive Young). Визуальный стиль не уступает международным конкурентам.

3. **Качественная карточка товара.** Содержит все элементы из research-чеклиста: бренд, название, цена старая/новая, процент скидки (бейдж), звёзды с количеством отзывов, wishlist, quick view, кнопка "В корзину" с анимацией при hover. Множественные бейджи (New + Хит) на одной карточке.

4. **Грамотная адаптивность.** Три брейкпоинта покрывают desktop, tablet, mobile. Сетки перестраиваются логично, hero меняет layout, корзина занимает весь экран на mobile, кнопки становятся full-width.

5. **Современные CSS/JS практики.** CSS-переменные, BEM-методология, IntersectionObserver для scroll-анимаций, backdrop-filter, aspect-ratio, CSS Grid -- код написан на уровне 2025-2026 стандартов.

---

## ТОП-5 слабых мест / что нужно доработать

1. **SEO полностью не готов.** Отсутствуют meta description, Open Graph, structured data (JSON-LD), favicon, canonical URL, alt-тексты. Без этих элементов сайт не будет индексироваться должным образом и не получит трафик из поиска. Это самый критичный недочёт.

2. **JS-функциональность на уровне прототипа.** Корзина статична (нет добавления/удаления/пересчёта), поиск не работает, quiz -- заглушка с alert(), quick view не реализован, wishlist не синхронизирован. Для MVP необходима рабочая корзина и поиск как минимум.

3. **Нет продвинутых фильтров.** Research выделяет фильтры по типу кожи, ингредиентам, бренду и цене как Приоритет 1. Текущие 4 кнопки (Все / Хиты / Новинки / Скидки) -- недостаточно для K-beauty магазина, где покупатель ищет решение проблемы.

4. **Нет реальных изображений товаров.** Emoji-заглушки создают впечатление незавершённости. Research единогласно указывает, что "продуктовое фото на белом фоне -- центр дизайна" у всех лидеров. Без реальных фото невозможно оценить восприятие дизайна в полной мере.

5. **Нет мультиязычности и интеграций.** Для казахстанского рынка критически важны: казахский язык, интеграция с Kaspi Pay / Kaspi Магазин, WhatsApp-чат для консультаций (упомянут в footer, но нет live-виджета). Research выделяет мультиязычность как Приоритет 2.

### Технические улучшения кода

- Привести BEM к строгому стандарту: заменить `.category-card h3` на `.category-card__title`.
- Убрать дублирование `top` в `.product-card__badge--best` (строка 427 style.css).
- Вынести hardcoded цвета из JS (`#e8458b`, `#4caf50`) в CSS-переменные или data-атрибуты.
- Добавить `preload` для шрифта Inter (критический ресурс).
- Разделить CSS на модули или использовать препроцессор (SCSS) при росте кодовой базы.
- Добавить `will-change: transform` на анимируемые элементы для GPU-оптимизации.
- Обернуть JS в DOMContentLoaded или использовать `defer` атрибут (сейчас скрипт в конце body -- работает, но `defer` надёжнее).

## Заключение

GlowSeoul -- качественный прототип, который стратегически верно позиционирован для казахстанского рынка. Структура страницы, выбор секций и визуальный стиль соответствуют лучшим мировым практикам K-beauty e-commerce.
