# API Reference

## Базовый URL

```
/api/v1
```

## Аутентификация

Все запросы (кроме `/auth`) требуют заголовок:

```
Authorization: Bearer <token>
```

---

## Эндпоинты

### Auth

| Метод | URL | Описание |
|-------|-----|----------|
| POST | `/auth/login` | Вход в систему |
| POST | `/auth/logout` | Выход |
| POST | `/auth/refresh` | Обновление токена |

### Транзакции

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/transactions` | Список транзакций (с фильтрами) |
| GET | `/transactions/:id` | Одна транзакция |
| POST | `/transactions` | Создать транзакцию |
| PUT | `/transactions/:id` | Обновить транзакцию |
| DELETE | `/transactions/:id` | Удалить (soft delete) |

**Параметры фильтрации** (`GET /transactions`):

| Параметр | Тип | Описание |
|----------|-----|----------|
| `from` | date | Начало периода |
| `to` | date | Конец периода |
| `category` | string | ID категории |
| `department` | string | ID отдела |
| `type` | string | `income` / `expense` |

### Бюджеты

| GET | `/budgets/:id` | Детали бюджета |

| PUT | `/budgets/:id` | Обновить бюджет |

### Отчёты

| Метод | URL | Описание |
|-------|-----|----------|
| GET | `/reports/pnl` | Отчёт о прибылях и убытках |

| GET | `/reports/balance` | Балансовый отчёт |
| GET | `/reports/department/:id` | Отчёт по отделу |

## Формат ответа

```json
{
  "success": true,
  "data": { ... },
  "meta": {
    "page": 1,
    "limit": 20,
    "total": 150
  }
}
```

## Коды ошибок

| Код | Описание |
|-----|----------|
| 400 | Некорректный запрос |
| 401 | Не авторизован |
| 403 | Нет доступа |
| 404 | Не найдено |
| 500 | Внутренняя ошибка сервера |
