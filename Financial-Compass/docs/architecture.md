# Архитектура Financial Compass

## Общая схема

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   Frontend   │────▶│   Backend    │────▶│  PostgreSQL   │
│   (React)    │◀────│  (Node.js)   │◀────│     DB        │
└──────────────┘     └──────────────┘     └──────────────┘
```

## Компоненты системы

### 1. Frontend (клиентская часть)

- **SPA** на React с маршрутизацией
- Компонентная архитектура
- Графики и дашборды через Recharts
- Адаптивный дизайн

### 2. Backend (серверная часть)

- REST API на Node.js / Express
- Аутентификация и авторизация (JWT)
- Валидация входящих данных
- Логирование операций

### 3. База данных

- PostgreSQL для хранения финансовых данных
- Миграции через Knex / Prisma
- Индексация для быстрых аналитических запросов

## Модель данных (ключевые сущности)

| Сущность | Описание |
|----------|----------|
| `Transaction` | Финансовая операция (доход/расход) |
| `Budget` | Бюджет отдела или проекта |
| `Category` | Категория расходов/доходов |
| `Department` | Отдел компании |
| `Report` | Сохранённый отчёт |
| `User` | Пользователь системы |

## Принципы

- Разделение ответственности (separation of concerns)
- Иммутабельность финансовых записей (только soft delete)
- Аудит всех изменений
- Ролевой доступ (admin, manager, viewer)
